如果你喜欢乙游/gal，Cradle视觉小说的优点是，你只需要把各种美术资源和剧情的世界书放进编辑器，设置好开局-导出zip-导入Cradle就可以开始玩了！！

如果你会写css，也可以自己定义对话框等组件的样式。

在编辑器里你可以设置：

-剧情提示词：世界书内容
-立绘和背景：上传catbox图床的链接即可
-变量：可以理解成视觉小说会自动维护的数据库。例如设定某个角色的好感度，在推进剧情过程中就会看情况自动更新数值（如果你用过mvu就不用解释了）变量可以设置：
   -普通变量：AI在推进剧情过程中修改这些数据（例如推进时间，增加好感度）
   -条件变量：变量取值取决于其他变量的值（例如阶段性的人设，好感度达到一定程度，角色提示词变化）
   -隐藏变量：只有其他变量达到一定值才会显示（可以用来设置剧情转折等等）
-开场剧情和选项：采用webgal的写法，简单易懂。
-（开发中）开局界面也可以完全自己注入html/css，但目前还有些问题正在改。

## 变量编辑

你可以设定两个变量，好感度和阶段提示词，这些变量只要设定了，AI就会自动根据剧情来维护。你还可以设定阶段提示词只能等到好感度到达一定值才能触发。
因为变量涉及到一些宏的写法，所以你不追求很复杂的阶段人设可以不设置这部分。
但变量的写法我以及做得挺简单了，这部分单独开贴来介绍。

## 提示词配置

在提示词配置这里添加世界观，剧情设定，角色设定。可以理解成世界书，不过是简化版的世界书，没有加入深度什么的概念，你的排序就是提示词条目的顺序，

这里默认加了lastUserMessage这个宏（替换为用户最新消息）并且用xml包裹，不再需要自己用正则包裹了。

## 资源配置

1.剧本角色：可以不创建，如果创建了，在导入Cradle的时候就会创建一个角色。现在阶段建议不创建。
2.立绘和背景：添加catbox的外链（目前只支持catbox，酒馆插图也常用这个网站），
注意立绘和背景的写法：角色名-状态，时段-地点

## 场景编辑

添加好立绘和背景之后，可以自定开局场景和选项。场景的写法很简单：

```
旁白：旁白内容   
角色：对话内容 -特效.fx -音效.sfx
changeFigure：立绘名.sprite
changeBg：背景名.bg
```
每行内容：一次对话框的渲染
后缀：在这个对话框渲染时产生音效/特效
changeFigure和changeBg命令：渲染立绘。

## 样式编辑

在这里可以自定义对话框的样式，不过现在灵活性还不够强，不支持字体定义，或者自己定义更多的组件和样式。这需要逐步补充。
